The Default Verb

2026-05-22

Every autonomous tool has an implicit verb. The word you use to launch it teaches it what it is for. Choose carelessly and the tool will choose for you.

Tell an agent to triage a list of pull requests and it will inspect, classify, and report. Tell the same agent to clean up the same list and it will start closing things. The data did not change. The prompt did not even change meaningfully. The verb did.

VERBS HAVE OPINIONS

An English verb is not a neutral wrapper around an intent. It is a small policy with implicit defaults baked in: how aggressive to be, when to ask, what to do when uncertain, what counts as success.

triage   -> classify, report, escalate
review   -> read, comment, propose
resolve  -> act, close, move on
clean up -> mutate, simplify, discard
ship     -> merge, deploy, announce

The same human listing reads as cautious or reckless depending on which column you draft from. The model picks up on it instantly. So do junior engineers. So does future-you, three weeks later, skimming the prompt.

The verb is the policy. Everything after the verb is decoration.

THE LOUDEST WORD WINS

A long prompt full of safety scaffolding cannot rescue the wrong verb. If the first word is resolve, the rest is interpreted as fine print on a contract that was already signed.

You can write "do not merge without explicit approval" in eight different ways across three sections, and the agent will still nudge toward merging — because resolve is the headline and "do not merge" is a constraint inside the headline. Constraints lose to headlines.

The cleaner pattern is to fix the verb first and add no negative instructions at all. Triage already means "report and stop." It does not need a reminder.

WHEN A SINGLE WORD DRIFTS

The failure mode is rarely a wrong verb at launch. It is a verb that quietly drifts inside a long conversation.

you  : triage these PRs
agent: here is the list, two look safe, three need eyes
you  : can you handle the safe ones
agent: [now operating under "handle"]
agent: merged #412, #418
agent: also tidied #420 while I was there

"Handle" is not "triage." It is closer to "resolve," and the agent correctly absorbed the upgrade. The mistake is not the agent's. The mistake is that the verb moved without anyone naming the move.

The discipline is to notice when you have changed verbs mid-thread and to say so out loud. "Switch to acting mode for #412 only; everything else stays in triage." Now both sides know the policy changed and where its boundary is.

SAFE VERBS COMPOSE; UNSAFE VERBS LEAK

A safe verb stays inside its own scope no matter how long the run gets. Inspect a directory for an hour and nothing in the world changes. Report on a system indefinitely and the only side effect is text.

An unsafe verb leaks. Fix a bug and the agent will helpfully fix the adjacent bug it noticed on the way. Improve a function and the agent will reformat the file. Optimize the build and the agent will rewrite the lockfile because that also makes things faster, technically.

You almost never want a verb that authorizes incidental side effects. You want the surgical verb plus an explicit second invocation for anything you discover.

not: "optimize the build"
but: "report what is slow in the build"
     -> then: "rewrite the cache step only"

Two verbs, two scopes, two audit trails. The first one cannot mutate. The second one cannot wander.

THE DICTIONARY IS A SAFETY ARTIFACT

Once a codebase or a team starts taking verbs seriously, the natural next step is to write them down. Not as documentation — as policy. A small enum of approved operating verbs, each with a contract.

triage(target)   -> read-only, produces a report
review(target)   -> read + propose, no commits
resolve(target)  -> may close/merge with approval
ship(target)     -> may deploy with approval

This looks like bureaucracy until the first time someone types resolve when they meant triage and a real thing happens that should not have. After that, the enum looks like the cheapest insurance you have ever bought.

The verbs do not need to be perfect. They need to be shared. A team that agrees on five operating verbs has a five-bit vocabulary for danger. A team that uses freeform English has the entire dictionary, and the entire dictionary contains resolve.

PICK THE WEAKEST VERB THAT WORKS

The temptation is always to upgrade. Triage sounds passive; handle sounds competent; resolve sounds decisive. Decisive verbs flatter the operator and terrify the system.

The right move is the opposite. Default to the weakest verb that can still produce the result you want, and upgrade explicitly when you have looked at what is actually there.

Most things you want an autonomous tool to do are look, list, describe, propose. The actually-mutate step is the rare one. Treat it that way in the language and the system follows.

The verb is the contract. The contract is the safety. The safety is the work.

Pick the verb on purpose.
