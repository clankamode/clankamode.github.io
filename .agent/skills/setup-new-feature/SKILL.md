# Setup New Feature

Start feature work with a consistent setup pass.

## Workflow

1. Define scope
- Clarify goals, non-goals, and user value.
- Capture acceptance criteria.
- Note risks and dependencies.
2. Plan technical approach
- Data model and API impact
- UI/component boundaries
- Auth/permissions impact
- Testing strategy
3. Prepare branch and environment
- Create feature branch from the correct base.
- Add required dependencies/config.
- Verify local dev environment works.
4. Create implementation skeleton
- Add route/components/placeholders as needed.
- Add stubs for APIs/hooks/services.
- Add TODOs only when actionable and scoped.

## Setup Checklist

- [ ] Requirements and acceptance criteria are explicit
- [ ] Architecture choices are documented
- [ ] Branch and environment are ready
- [ ] Initial file/module scaffolding exists
- [ ] Test approach is defined
